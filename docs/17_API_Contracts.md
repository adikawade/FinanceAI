# FinanceAI — API Contracts

**Document Version:** 1.0.0

## API Versioning
All public APIs must use explicit versioning (e.g. /api/v1).

## Core Principles
- Stable contracts
- Backward compatibility
- Standard request/response DTOs
- Structured error responses
- Authentication and authorization on every protected endpoint

## Primary Interfaces
- Authentication APIs
- Transaction APIs
- Budget APIs
- Investment APIs
- Tax APIs
- AI APIs
- Admin APIs
- CA APIs

## Security
- Firebase Authentication
- Firebase App Check
- HTTPS only
- JWT validation
